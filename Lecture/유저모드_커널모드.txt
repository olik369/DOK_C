장치를 추상화한 장치파일(Device Operation)에 할당된 I/O버퍼에 내가 입력한 키보드 값이 들어가있음? 으로 이해하면 될까??


ex) C에서 printf를 통해 모니터에 출력하는 과정을 보자
Process(코드를 통해 Application부분에서 실행됨) -> API(application programming interface) 는 File Interface(파일 인터페이스, 장치를 추상화한 파일임)를 통해 시스템콜 호출함
-> 커널모드의 구성요소로 가게됨 [이때 커널모드로 변경됨(시스템 콜을 통해!!)] -> Device Driver -> 인터럽트 요청(IRQ) -> CPU는 잠시 모니터와 통신 후 모니터에 출력
->  완료했다는 인터럽트를 다시 Device Driver에 보냄
OS
- User Mode

- Kernal Mode(운영체제의 핵심)
시스템의 전반을 관리/감독하는 역할, 하드웨어와 관련된 작업을 직접 수행
for 시스템 보호

- OS상에서 프로그램이 실행되는 과정
프로그램의 현재 CPU상태를 저장 -> 인터럽트나 시스템콜을 직접처리(즉, CPU에서 커널코드가 실행됨)
-> 처리 완료 후, 중단됐던 프로그램의 CPU상태를 복원 -> 다시 통제권을 User Mode에게 넘겨줌 -> User Mode에서 프로그램이 이어서 실행됨

- Interrupt
시스템에서 발생한 다양한 종류의 이벤트 혹은 그런 이벤트를 알리는 메커니즘
ex) 전원에 문제가 생겼을 때, I/O 작업이 완료됐을 때, 시간이 다 됐을때(timer 관련), 0으로 나눴을때. 잘못된 메모리 공간에 접근을 시도할때

- System Call
프로그램이 OS커널이 제공하는 서비스를 이용하고 싶을 때 시스템 콜을 통해 실행
ex) 프로세스/스레드 관련, 파일 I/O관련, 소켓 관련, 장치 관련, 프로세스 통신 관련
